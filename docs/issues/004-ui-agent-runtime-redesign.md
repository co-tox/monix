# Issue: UI 개선 — 에이전트 런타임 스타일로 전면 개편

## 요약

현재 단순 텍스트 출력 방식의 UI를 Claude Code 스타일의 에이전트 런타임 느낌으로 개선합니다.
단순 라벨+값 나열에서 벗어나 도구 호출, 상태 전환, 결과 렌더링이 시각적으로 구분되는
인터랙티브 CLI로 발전시킵니다.

## 현재 문제

- 환영 화면과 응답이 박스/텍스트로만 구성되어 에이전트 동작감이 없음
- 명령 실행 중 아무런 시각적 피드백이 없음 (로딩 스피너 등 부재)
- AI 응답과 시스템 메시지가 동일한 스타일로 출력되어 구분이 어려움
- `/ask` 호출 시 응답 대기 중 터미널이 멈춘 것처럼 보임

## 변경 목표

### 1. 도구 호출 시각화 (Tool Use Indicator)

명령 실행 시 Claude Code처럼 어떤 도구가 실행되는지 표시합니다.

```
  ⏺  snapshot 수집 중...
  ✔  snapshot  (0.3s)
```

- `⏺` : 실행 중 (애니메이션 또는 정적 심볼)
- `✔` : 완료
- `✖` : 실패
- 실행 시간을 괄호 안에 표기

### 2. 스피너 (Spinner)

Gemini/Claude API 호출 대기 중 스피너를 표시합니다.

```
  ⠋ Gemini에 질문 중...
```

- `itertools.cycle`로 `⠋⠙⠹⠸⠼⠴⠦⠧⠇⠏` 프레임 순환
- 응답 수신 후 스피너 지우고 결과 출력
- 스레드 기반으로 메인 루프를 차단하지 않음

### 3. 응답 헤더 구분

AI 응답, 시스템 메시지, 경고를 시각적으로 구분합니다.

```
◆ Gemini                          # AI 응답 헤더
● 시스템                           # 시스템 메시지
⚠ 경고                            # 경고
```

### 4. 메트릭 바 개선

현재 `[####----]` 방식에서 더 세밀한 블록 문자 기반 바로 변경합니다.

```
CPU   ████████████░░░░░░░░░░░░  48.2%
Mem   ██████████████████░░░░░░  72.5%  1.2 GiB free
Disk  ██████░░░░░░░░░░░░░░░░░░  25.1%  80 GiB free
```

- 채운 블록: `█`, 빈 블록: `░`
- 임계치 초과 시 채운 블록 색상이 yellow → red로 변경

### 5. 명령 결과 패널

각 명령 결과를 제목이 있는 패널로 렌더링합니다.

```
┌─ /top ────────────────────────────────────────┐
│  PID      CPU%   MEM%   COMMAND               │
│  1234     12.3    4.1   python3               │
│   891      8.7    1.2   node                  │
└───────────────────────────────────────────────┘
```

- `┌┐└┘─│` 박스 드로잉 문자 사용
- 현재 `+---+` 방식에서 교체

### 6. 상태 라인 (Status Line)

REPL 프롬프트 위에 현재 컨텍스트 정보를 표시합니다.

```
  host: app-1  cpu: 12.3%  mem: 48.2%  [Gemini ✔]
monix >
```

## 수정 대상 파일

- `monix/render.py` — 전반적인 렌더링 함수 개선 (메인 작업)
- `monix/cli.py` — 스피너 통합, 도구 호출 래퍼 추가

## 구현 순서 (권장)

1. 메트릭 바 블록 문자 교체 (`_bar()`)
2. 박스 드로잉 문자로 패널 변경 (`_rule()`, `render_panel()`)
3. 응답 헤더 구분 (`render_reply()`)
4. 도구 호출 시각화 래퍼
5. 스피너 구현 및 `/ask` 연동
6. 상태 라인 추가

## 완료 기준

- `monix` 실행 시 에이전트 런타임 느낌의 환영 화면 출력
- `/ask` 호출 시 스피너가 표시되고 응답 후 사라짐
- 도구 실행 시 `⏺ ... ✔` 흐름이 보임
- 기존 테스트 전체 통과
- `NO_COLOR=1` 환경에서도 정상 동작 (ANSI 없이 심볼만 표시)
